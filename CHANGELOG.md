# 변경 기록

## V1.0.7
- 대시보드에 실제로 동작하는 프로젝트 액션 버튼을 추가했습니다.
- 최근 프로젝트 카드에서 주제, 리서치, 대본, 장면, 렌더 화면으로 바로 이동할 수 있게 했습니다.
- 대시보드에서 프로젝트 전체 실행과 완료 영상 열기가 가능하도록 연결했습니다.
- 완료, 진행중, 실패, 업로드 대기 수를 한눈에 보이도록 정리했습니다.

## V1.0.6
- 좌측 워크플로우에 각 단계별 `완료 / 진행중 / 대기 / 실패` 상태가 실제 프로젝트 진행 상태 기준으로 표시되도록 수정했습니다.
- 단계 번호 카드와 상태 텍스트에 색을 나눠서 현재 어디서 멈췄는지 바로 보이도록 조정했습니다.
- Google Vids는 공식 도움말상 Vids 내부 Gemini 이미지 생성은 가능하지만, 현재 앱에서 직접 붙일 공식 API 경로는 확인되지 않아 이번 버전에는 연결하지 않았습니다.

## V1.0.5
- n8n이 프로젝트 상태를 읽고 단계별 실행을 호출할 수 있도록 에이전트 전용 API를 추가했습니다.
- `n8n/longform-ai-production-pipeline.json` 파일에 주제 도출, 리서치, 대본, 렌더, 최종 AI 요약까지 이어지는 새 워크플로우를 추가했습니다.
- `docker-compose.yml`에 로컬 앱 주소와 OpenAI 환경 변수를 연결해 n8n 워크플로우에서 키를 직접 노출하지 않도록 정리했습니다.
- 로컬 `.env`에 API 키 파일 기준 OpenAI 키를 반영하고, n8n 인스턴스에 새 워크플로우를 import했습니다.

## V1.0.4
- 주제 저장 시 프롬프트 원문과 실제 사용하는 도출 주제를 분리했습니다.
- 리서치 자동 수집과 대본 생성이 프롬프트 문장을 그대로 쓰지 않고 도출된 주제 기준으로 동작하도록 수정했습니다.
- 프로젝트 생성 화면과 프로젝트 상세 화면에서 주제 프롬프트 입력과 도출된 주제 표시를 분리했습니다.
- 영문 지시형 프롬프트에서 채널명이나 지시문이 그대로 노출되지 않도록 fallback 주제 도출 로직을 보정했습니다.

## V1.0.3
- 주제 설정을 짧은 제목 입력이 아니라 주제 프롬프트 입력 방식으로 변경했습니다.
- 프로젝트 생성과 주제 설정에서 영상 길이를 선택할 수 있게 하고, 대본 자동 생성이 그 길이를 기준으로 동작하도록 맞췄습니다.
- 리서치 저장 후 저장 안내와 저장된 리서치 내용을 같은 화면에서 바로 확인할 수 있게 했습니다.
- 전체 폰트와 카드 크기를 조정해 가독성을 높였습니다.

## V1.0.2
- 전체 기본 폰트 크기와 입력창, 버튼, 카드 제목 크기를 키웠습니다.
- 좌측 사이드바 폭과 워크플로우 목록 가독성을 높였습니다.

## V1.0.1
- 버전 규칙을 `V1.0.x` 형식으로 변경했습니다.
- 홈과 프로젝트 상세 화면을 다크모드 기준의 스튜디오형 UI로 재구성했습니다.
