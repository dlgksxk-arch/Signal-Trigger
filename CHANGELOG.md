# 변경 기록

## V1.0.19
- `package.json` 버전과 화면 버전 파일을 다시 맞춰, 파일 기준 버전이 서로 다르게 보이던 문제를 정리했습니다.
- 서버를 `node --watch src/server.js` 직접 실행 방식으로 다시 띄워, 최신 코드 응답이 안정적으로 보이게 맞췄습니다.

## V1.0.18
- 서버 시작 스크립트를 감시 모드로 바꿔, 코드 변경 후 서버 버전이 예전 값으로 남는 문제를 줄였습니다.
- `npm run start` 실행 시 파일 수정 후 서버가 자동으로 다시 읽히도록 맞췄습니다.

## V1.0.17
- 홈 화면에서 `noticeMessage` 값이 없는 경우에도 에러 없이 렌더되도록 방어 코드를 추가했습니다.
- 프로젝트 삭제 기능 추가 이후 대시보드 진입 시 발생하던 `noticeMessage is not defined` 오류를 막았습니다.

## V1.0.15
- 렌더 단계에서 장면 이미지에 미세한 패닝과 줌 느낌이 생기도록 움직임 필터를 추가했습니다.
- 주제 도출과 리서치 수집에서 역사, 전쟁, 제국, 조약, 국경, 혁명 같은 키워드를 더 우선적으로 잡도록 보정했습니다.
- 현재 이 버전부터는 역사성과 배경 설명이 강한 주제를 먼저 고르도록 검색 순서를 조정했습니다.

## V1.0.14
- 장면 이미지 생성 모델을 기본 `Pollinations flux`에서 `OpenAI 이미지 생성 모델` 우선 사용으로 전환했습니다.
- 현재는 OpenAI 키가 있으면 `gpt-image-1.5`와 `images/generations` 경로를 먼저 사용하고, 실패 시에만 기존 Pollinations와 로컬 fallback으로 내려갑니다.
- `.env.example`에 이미지 모델과 품질 설정 값을 추가했습니다.

## V1.0.13
- 대본 생성에서 음악 큐와 구간 라벨이 나오지 않도록 프롬프트와 후처리를 강화했습니다.
- 장면 프롬프트에 인물 클로즈업 남발 방지와 다양한 시각 소재 규칙을 추가해 지도, 시설, 국경, 전장, 문서, 항만, 시위, 외교 장면 등을 더 적극적으로 쓰도록 보정했습니다.
- 더빙 생성 시 `edge_tts`가 직접 만든 자막 파일을 같이 사용하도록 바꿔 자막과 음성 타이밍이 맞도록 수정했습니다.

## V1.0.12
- 썸네일이 제목 고정 방식이 아니라 대본과 장면 중 가장 후킹한 문구와 대표 장면을 골라 굵은 텍스트 스타일로 합성되도록 변경했습니다.
- 대본 생성 프롬프트와 후처리를 수정해 `인트로`, `본론`, `결론` 같은 구분 라벨이 나오지 않도록 보정했습니다.
- 더빙 기본 음성을 40대 초반 느낌의 남성 톤 기준으로 한국어, 영어, 일본어 모두 남성 보이스로 변경했습니다.

## V1.0.11
- 장면 탭 카드를 썸네일 중심으로 정리하고, 장면 번호만 오버레이로 표시하도록 변경했습니다.
- 초기화 버튼에 확인 창을 추가하고, 초기화 후 현재 보고 있던 단계 화면으로 돌아오도록 수정했습니다.
- 초기화 함수가 리서치, 대본, 장면, 렌더 결과와 생성 폴더를 실제로 비우는지 임시 프로젝트 기준으로 검증했습니다.

## V1.0.10
- 장면 생성 프롬프트에 지정하신 영문 편집 일러스트 지시문을 기본 베이스로 먼저 넣도록 변경했습니다.
- 각 장면은 이제 베이스 프롬프트 뒤에 해당 장면 내레이션, 주제, 톤, 화면 비율, 색상 정보를 붙여 생성됩니다.

## V1.0.9
- 각 단계에 완료, 진행중, 중지, 대기, 실패 상태를 실제 자동 생성 진행상황 기준으로 더 직관적으로 표시하도록 보정했습니다.
- 프로젝트 상세 화면에 전체 자동 생성 진행 카드와 진행률 바를 추가하고, 시작, 일시중지, 재개, 초기화 버튼을 연결했습니다.
- 자동 생성이 실행 중이거나 중지 요청 중일 때 화면이 자동 새로고침되도록 추가했고, 기존 완료 프로젝트도 잘못된 대기 상태로 보이지 않게 정리했습니다.

## V1.0.8
- 장면 화면에서 가로형 프로젝트가 세로처럼 보이던 표시 문제를 수정했습니다.
- 장면 카드를 썸네일 중심 구조로 바꾸고, 클릭 시 확대 이미지와 프롬프트를 보는 모달을 추가했습니다.
- 모달 안에서 해당 장면만 바로 다시 생성할 수 있게 연결했습니다.

## V1.0.7
- 대시보드에 실제로 동작하는 프로젝트 액션 버튼을 추가했습니다.
- 최근 프로젝트 카드에서 주제, 리서치, 대본, 장면, 렌더 화면으로 바로 이동할 수 있게 했습니다.
- 전체 실행과 완료 영상 열기를 대시보드에서 바로 실행할 수 있게 연결했습니다.
- 완료, 진행중, 실패, 업로드 대기를 상단에서 바로 보이게 정리했습니다.

## V1.0.6
- 좌측 워크플로우에 각 단계별 완료, 진행중, 대기, 실패 상태를 실제 프로젝트 진행 기준으로 표시하도록 수정했습니다.
- 단계 번호 카드와 상태 텍스트 색을 나눠서 현재 어디서 멈췄는지 바로 보이게 조정했습니다.
- Google Vids는 공식 API 경로를 확인하지 못해 이번 버전에는 연결하지 않았습니다.

## V1.0.5
- n8n에서 프로젝트 상태를 읽고 단계별 실행을 호출할 수 있도록 앱 전용 API를 추가했습니다.
- `n8n/longform-ai-production-pipeline.json`에 주제 도출, 리서치, 대본, 렌더, AI 요약 흐름을 추가했습니다.
- `docker-compose.yml`과 로컬 `.env`를 통해 OpenAI 환경 변수를 n8n에 연결했습니다.

## V1.0.4
- 주제 프롬프트 원문과 실제 작업에 쓰는 도출 주제를 분리했습니다.
- 리서치 자동 수집과 대본 생성이 프롬프트 문장 자체가 아니라 도출된 주제 기준으로 동작하도록 수정했습니다.
- 프로젝트 생성 화면과 상세 화면에 주제 프롬프트와 도출 주제를 분리해 표시했습니다.
- 영어 지시형 프롬프트도 그대로 제목으로 남지 않도록 fallback 주제 도출 로직을 보정했습니다.

## V1.0.3
- 주제 설정을 짧은 제목 입력이 아니라 주제 프롬프트 입력 방식으로 변경했습니다.
- 프로젝트 생성과 주제 설정에서 영상 길이를 선택할 수 있게 하고, 대본 생성이 그 길이를 기준으로 동작하도록 맞췄습니다.
- 리서치 저장 안내와 저장된 리서치 내용을 같은 화면에서 바로 확인할 수 있게 했습니다.
- 전체 폰트와 카드 크기를 조정해 가독성을 높였습니다.

## V1.0.2
- 전체 기본 폰트 크기와 입력창, 버튼, 카드 제목 크기를 키웠습니다.
- 좌측 사이드바와 워크플로우 목록 가독성을 높였습니다.
