# 변경 기록

## V1.0.2
- 전체 기본 글자 크기와 입력창, 버튼, 카드 제목 크기를 한 단계씩 올렸습니다.
- 좌측 사이드바 폭을 넓히고 메뉴 패딩과 단계 번호 크기를 키워 워크플로우가 더 잘 보이도록 조정했습니다.
- 좌측 워크플로우 카드의 목록 글자 색과 크기를 올려 실제 읽히도록 수정했습니다.

## V1.0.1
- 버전 규칙을 `V1.0.x` 형식으로 변경했습니다.
- 홈과 프로젝트 상세 화면을 유튜브 스튜디오 느낌의 다크 UI 구조로 재구성했습니다.
- 좌측 사이드바, 상단 상태 영역, 카드형 작업 패널, 단계 편집 화면의 타이포와 간격을 다시 정리했습니다.

## V0.0.7
- 홈 좌측 메뉴를 실제 페이지 경로로 분리해서 대시보드, 새 프로젝트, 채널, 프로젝트 목록 화면이 각각 열리도록 바꿨습니다.
- 프로젝트 단계 탭도 실제 단계 경로를 가지도록 정리하고, 같은 화면 안에서는 즉시 전환되도록 유지했습니다.
- 전체 UI의 글자 크기, 카드 간격, 사이드바 폭, 버튼 크기, 요약 카드 구성을 다시 정리했습니다.

## V0.0.6
- 좌측 워크플로우 탭이 페이지 새로고침 없이 바로 전환되도록 실제 탭 동작을 추가했습니다.
- 단계 탭을 누르면 오른쪽 세부 패널과 현재 단계 표시가 함께 바뀌도록 맞췄습니다.
- 좌측 바에 단계 번호와 짧은 단계명을 같이 보여주도록 UI를 조정했습니다.

## V0.0.5
- 좌측 워크플로우 영역을 더 얇은 세로 탭 바 형태로 다시 조정했습니다.
- 단계 탭을 세로 스택으로 배치하고, 선택된 탭 상세가 오른쪽 패널에 보이도록 구조를 더 분명하게 맞췄습니다.

## V0.0.4
- Dribbble Trior 대시보드 방향을 참고해 메인 화면과 상세 화면을 대시보드형 UI로 재구성했습니다.
- 좌측 고정 사이드바, 상단 요약 영역, 카드형 입력 화면, 상태 카드 구성을 적용했습니다.
- 각 화면에서 워크플로우 진행 구조가 보이도록 정보 배치를 다시 정리했습니다.

## V0.0.3
- 현재 작업 디렉터리를 `REPO` Git 저장소와 연결했습니다.
- 앞으로 버전업 기준으로 이 저장소를 사용하도록 맞췄습니다.
- 아직 원격 저장소 URL이 없어 GitHub 푸시는 불가능한 상태입니다.

## V0.0.2
- 버전 규칙을 `major.minor.patch`로 고정했습니다.
- 루트에 버전별 변경사항 기록 파일을 추가했습니다.
- 실행 주소 안내를 `http://127.0.0.1:3100` 기준으로 수정했습니다.
- 서버 창을 닫으면 사이트가 닫힌다는 점을 문서에 명시했습니다.
- GitHub 인증은 확인됐지만 원격 저장소 연결은 없는 상태로 기록했습니다.

## V0.0.1
- Longform 자동 공장 기본 프로젝트를 생성했습니다.
- 주제, 리서치, 대본, 장면, 렌더, 업로드 흐름의 MVP를 추가했습니다.
- `n8n` 연동 예시와 영상 렌더 파이프라인을 포함했습니다.
